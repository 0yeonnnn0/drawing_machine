# 디자인 브리핑 — 낙서 공유 앱

## 서비스 소개

그림을 그려서 링크로 공유하면, 받은 사람이 같은 캔버스에서 같이 그릴 수 있는 협업 드로잉 웹앱(PWA).

---

## 디자인 컨셉

**Undertale UI 스타일의 레트로 픽셀 디자인**

"투박하고 옛날 그림판 같은 느낌"을 의도한 앱입니다.
앱의 핵심 기능(낙서, 공유)과 비주얼이 직결되도록 레트로 감성을 전면에 내세웁니다.

### 레퍼런스
- **메인**: Undertale UI
- **서브**: Windows 95 / MS Paint

---

## 디자인 가이드라인 (확정)

### 타이포그래피
- **일반 폰트** 사용 (픽셀 폰트 X)
- UI 요소(버튼, 테두리 등)만 픽셀 스타일로 처리
- 가독성 우선

### 컬러 팔레트
- **옛날 16색 팔레트 느낌**으로 구성
- 원색 중심, 색 수 최소화
- 흰 배경 기본
- 포인트 컬러 1~2개
- 참여자 커서 구분용 원색 6~8가지 (팔레트 내에서)

### UI 컴포넌트
- 버튼 / 모달 / 카드 / 입력창 **전부 픽셀 테두리 스타일**
- Windows 95 오마주: 굵은 검정 테두리, 그림자 없음
- 선택 / 포커스 상태: 픽셀 커서 인디케이터 (화살표 or 하트)
- 호버: 부드러운 트랜지션 없이 즉각 상태 변환 (툭툭 끊기는 느낌)
- 스크롤바도 픽셀 스타일 권장

### 아이콘
- **기성 픽셀 아이콘셋** 활용 (Game Icons / Kenney 등)
- 16×16 or 32×32 기반

### 애니메이션
- CSS `steps()` 기반 — 부드럽지 않고 툭툭 끊기게
- 로딩 인디케이터: 도트 애니메이션 (···)
- 페이지 전환 트랜지션 최소화

---

## 페이지 목록 (와이어프레임 필요)

총 6개 페이지. 각 페이지별 **모바일 + 데스크톱** 레이아웃 모두 필요합니다.

### 1. 랜딩 페이지 `/`
- GNB (로그인/로그아웃 상태 분기)
- Hero 섹션: 서비스 소개 + CTA 버튼
- 핵심 기능 소개 섹션 (그리기 / 공유 / 협업)
- **상태**: 비로그인 CTA → "시작하기" / 로그인 CTA → "내 캔버스 보기"

### 2. 로그인 페이지 `/login`
- 서비스 로고 + 짧은 카피
- Google 로그인 버튼 (단일)
- 심플하게

### 3. 대시보드 `/dashboard`
- GNB (프로필, 로그아웃)
- "새 캔버스 만들기" 버튼 (눈에 띄게)
- 캔버스 카드 그리드
  - 썸네일 이미지
  - 제목
  - 최근 수정일
  - 공개/비공개 뱃지
  - 컨텍스트 메뉴: 열기 / 링크 복사 / 삭제
- **빈 상태(empty state)** 별도 디자인 필요

### 4. 초대 링크 진입 `/join/[token]`
- 캔버스 제목 미리보기
- 닉네임 입력 필드 (자동생성값 표시 + 수정 가능)
- "참여하기" 버튼
- "로그인으로 참여하기" 텍스트 링크
- **에러 상태**: 유효하지 않은 토큰 안내

### 5. 캔버스 에디터 `/canvas/[id]` ← 가장 중요
**MS Paint 레이아웃 오마주**

- **헤더**
  - 캔버스 제목 (owner: 클릭해서 편집 / guest: 읽기 전용)
  - 참여자 목록 (온라인 인디케이터 포함)
  - 링크 복사 버튼 (owner만 표시)
  - PNG 다운로드 버튼
  - 공개/비공개 토글 (owner만)

- **툴바** (데스크톱: 좌측 세로 / 모바일: 하단 가로)
  - 펜 도구
  - 지우개 도구
  - 색상 팔레트 (16색)
  - 굵기 조절
  - 이미지 삽입
  - 텍스트 삽입
  - Undo 버튼

- **캔버스 영역**
  - 드로잉 캔버스 (최대 영역 차지)
  - 다른 참여자 커서 + 닉네임 라벨 (원색으로 각각 구분)
  - 커스텀 픽셀 커서

- **토스트 / 시스템 메시지**
  - 참여자 입장 / 퇴장 알림
  - 연결 끊김 → 재연결 시도 인디케이터
  - owner가 캔버스 삭제 시 참여자 전원 안내

### 6. 에러 페이지 `/404`, `/error`
- 에러 메시지 (픽셀 스타일로 재미있게)
- "홈으로 돌아가기" 버튼
- 케이스: 없는 페이지 / 만료된 링크 / 권한 없음

## 산출물 요청

- 각 페이지 와이어프레임 (모바일 + 데스크톱)
- 디자인 시스템: 컬러 팔레트 / 타이포그래피 스케일 / 컴포넌트 (버튼, 입력창, 모달, 카드, 뱃지, 토스트)
- 캔버스 에디터 상세 UI (툴바 상태별, 참여자 커서 예시)
- 빈 상태(empty state) 일러스트 or 픽셀 아트

---

## 참고사항

- PWA — 브라우저에서 바로 실행, 홈 화면 추가 가능
- 반응형 필수 (모바일 우선 권장)
- 캔버스 에디터 툴바 위치가 모바일/데스크톱에서 달라짐 — 레이아웃 분기 명확히
- 개발 프레임워크: Next.js + Tailwind CSS 예정 → 디자인 토큰(색상, 간격, 타이포) Figma Variables로 정리해주시면 협업 편합니다
